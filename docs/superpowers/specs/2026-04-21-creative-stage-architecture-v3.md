# Creative Stage — Architecture v3 (Livepeer-First)

> **Primary mode:** Remote via Livepeer network (SDK → BYOC Orch → fal Scope runner)
> **Secondary mode:** Local Scope backend (optional, for development)
> **Hard constraint:** Zero storyboard regression. Feature branch only.

---

## 1. The Reality Check: How Scope Streams Work Today

### Remote Livepeer Mode (PRIMARY — production)

```
Browser
  → SDK Service (sdk.daydream.monster)
    POST /stream/start { model_id: "scope", params: { prompt, graph, ... } }
    → BYOC Orch (go-livepeer)
      → Scope Orch (orch-staging-1/2.daydream.monster)
        → fal.ai Scope runner (WebSocket: wss://fal.run/daydream/scope-livepeer/ws)
          → Scope backend spawns inside fal container
          → longlive pipeline processes frames on GPU
  ← Returns: stream_id, trickle channel URLs

Browser publishes input frames:
  POST /stream/{id}/publish?seq=N  (JPEG blob, 10fps)

Browser polls output frames:
  GET /stream/{id}/frame  (JPEG blob, polls at 200ms)

Browser controls mid-stream:
  POST /stream/{id}/control  { type: "parameters", params: { prompts: "new prompt" } }
```

**This is HTTP trickle protocol — NOT WebRTC.**
The fal runner doesn't expose WebRTC. There's no peer connection.

### Local Scope Mode (SECONDARY — development only)

```
Browser → Local Scope backend (http://localhost:8000)
  → WebRTC peer connection (SDP offer/answer)
  → Pipeline processes frames on local GPU
  → WebRTC output stream (video + audio)
```

This uses Scope's `useUnifiedWebRTC` hook. Only works with a local Scope install.

### What This Means for Creative Stage

The v2 architecture assumed WebRTC everywhere. **Wrong.** For production (Livepeer mode), we use the SDK's trickle protocol — same as storyboard. But we improve the playback.

## 3. Improved Stream Playback (The Key Fix)

### Problem: Storyboard's Current Player

```
Poll /stream/{id}/frame every 200ms
  → Returns JPEG blob
  → Create blob URL → set <img>.src
  → Revoke previous blob URL
```

Issues:
- **Visible stutter** at 5fps (200ms intervals)
- **No audio** (JPEG frames only)
- **Memory pressure** (blob URL churn)
- **No smooth transitions** between frames

### Solution: Canvas-Based Frame Player

```typescript
// packages/scope-player/src/TrickleVideoPlayer.tsx

// Instead of <img src={blobUrl}>, use a <canvas> element:
// 1. Decode JPEG blob into ImageBitmap (off main thread)
// 2. Draw to canvas with requestAnimationFrame (smooth)
// 3. Hold last frame if poll is slow (no flicker)
// 4. Double-buffer: decode next frame while displaying current
```

```tsx
export function TrickleVideoPlayer({ streamId, sdkUrl, apiKey, onFrame }) {
  const canvasRef = useRef<HTMLCanvasElement>(null);
  const [fps, setFps] = useState(0);

  useEffect(() => {
    const ctx = canvasRef.current?.getContext("2d");
    if (!ctx) return;

    let running = true;
    let lastBitmap: ImageBitmap | null = null;
    let frameCount = 0;
    const fpsStart = performance.now();

    async function pollLoop() {
      while (running) {
        try {
          const resp = await fetch(`${sdkUrl}/stream/${streamId}/frame`, {
            headers: apiKey ? { Authorization: `Bearer ${apiKey}` } : {},
          });
          if (resp.ok && resp.headers.get("content-type")?.includes("image")) {
            const blob = await resp.blob();
            const bitmap = await createImageBitmap(blob);
            lastBitmap = bitmap;
            frameCount++;
          }
        } catch { /* network hiccup — hold last frame */ }
        await new Promise(r => setTimeout(r, 100)); // 10fps poll
      }
    }

    function renderLoop() {
      if (!running) return;
      if (lastBitmap && ctx) {
        ctx.drawImage(lastBitmap, 0, 0, ctx.canvas.width, ctx.canvas.height);
      }
      // Update FPS counter
      const elapsed = (performance.now() - fpsStart) / 1000;
      if (elapsed > 1) setFps(Math.round(frameCount / elapsed));

      requestAnimationFrame(renderLoop);
    }

    pollLoop();
    renderLoop();
    return () => { running = false; };
  }, [streamId, sdkUrl, apiKey]);

  return <canvas ref={canvasRef} style={{ width: "100%", height: "100%" }} />;
}
```

**Improvements over storyboard's `<img>` approach:**
- **No flicker** — canvas holds last frame while next loads
- **Smooth** — requestAnimationFrame for display, async fetch for data
- **Off-thread decode** — `createImageBitmap` decodes JPEG without blocking UI
- **No blob URL leaks** — ImageBitmap doesn't use blob URLs
- **FPS counter** — visible feedback on stream health

### Audio in SDK Mode

The Scope pipeline can generate audio (via Seedance, etc.), but the trickle protocol only carries JPEG frames. Audio options:

**Option A: Separate audio poll** — SDK adds `/stream/{id}/audio` endpoint that returns audio chunks. Creative Stage polls and plays via `<audio>` element.

**Option B: Scope's WebSocket events** — The fal runner sends audio data via the WebSocket connection to the SDK. SDK buffers and exposes via a streaming endpoint.

**Option C: Client-side music** — For v1, audio comes from the user's music track (dropped onto canvas), not from the stream itself. The `/mix` command combines stream recording + music at the end.

**Decision for v1:** Option C. User-provided music synced via modulation. Stream audio is a future enhancement when the SDK adds audio channels.

## 4. Stream Control Flow (Remote Livepeer)

### Starting a Stream

```
User: "a dreamy forest with fireflies"

Agent calls stage_start tool:
  → POST sdk.daydream.monster/stream/start
    { model_id: "scope", params: {
        prompts: "a dreamy forest with fireflies, cinematic",
        graph: { nodes: [...], edges: [...] },  // text-only template
        pipeline_ids: ["longlive"],
        noise_scale: 0.7,
        denoising_step_list: [1000, 750, 500, 250],
    }}
  ← { stream_id: "abc123" }

ScopePlayer starts:
  → Polls /stream/abc123/frame every 100ms
  → Renders frames to canvas
  → Shows "Connecting to fal runner..." during cold start
```

### Changing Prompt Mid-Stream (No Restart)

```
User: "add rain and neon reflections"

Agent calls stage_prompt tool:
  → POST sdk.daydream.monster/stream/abc123/control
    { type: "parameters", params: {
        prompts: "dreamy forest with fireflies, rain, neon reflections, cinematic",
    }}
  ← 200 OK

Stream morphs seamlessly — Scope's prompt-traveling handles the transition.
```

### VACE Reference Image

```
User drags a photo onto the canvas near LiveCard.

stage_reference tool:
  → Upload image to GCS via /api/upload → get public URL
  → POST sdk.daydream.monster/stream/abc123/control
    { type: "parameters", params: {
        vace_enabled: true,
        vace_ref_images: ["https://storage.googleapis.com/.../ref.jpg"],
        vace_context_scale: 0.8,
    }}

Stream adopts the reference image's colors/structure.
```

### Modulation (Beat Sync)

```
User drops music, agent detects 85 BPM.

stage_sync tool:
  → POST sdk.daydream.monster/stream/abc123/control
    { type: "parameters", params: {
        modulation: {
          noise_scale: { enabled: true, shape: "cosine", rate: "bar", depth: 0.3 }
        }
    }}

Stream noise oscillates with the beat.
```

### Recording

```
User: "record this"

stage_record tool:
  → POST sdk.daydream.monster/stream/abc123/control
    { type: "parameters", params: { recording: true } }

  ... scenes play ...

  → POST sdk.daydream.monster/stream/abc123/control
    { type: "parameters", params: { recording: false } }

  → GET recording result (URL to recorded file)
  → Mix with music via creative-kit mixMedia()
  → Final video card on canvas
```

## 5. Agent Tools — SDK-First

```typescript
// lib/stage-tools.ts

export function createStageTools(sdkUrl: string, apiKey: string, streamIdRef: { current: string | null }) {
  const headers = () => ({
    "Content-Type": "application/json",
    ...(apiKey ? { Authorization: `Bearer ${apiKey}` } : {}),
  });

  return [
    {
      name: "stage_start",
      description: "Start a live AI video stream from a scene description",
      parameters: { prompt: "string", preset: "string?", graph_template: "string?" },
      async execute(args) {
        const graph = buildGraph(args.graph_template || "text-only");
        const resp = await fetch(`${sdkUrl}/stream/start`, {
          method: "POST", headers: headers(),
          body: JSON.stringify({
            model_id: "scope",
            params: { prompts: args.prompt, graph, pipeline_ids: ["longlive"], noise_scale: 0.7 },
          }),
        });
        const data = await resp.json();
        streamIdRef.current = data.stream_id;
        return JSON.stringify({ stream_id: data.stream_id, status: "started" });
      },
    },

    {
      name: "stage_prompt",
      description: "Update the live stream prompt (morphs seamlessly, no restart)",
      parameters: { prompt: "string", noise_scale: "number?" },
      async execute(args) {
        if (!streamIdRef.current) return JSON.stringify({ error: "No active stream" });
        await fetch(`${sdkUrl}/stream/${streamIdRef.current}/control`, {
          method: "POST", headers: headers(),
          body: JSON.stringify({ type: "parameters", params: { prompts: args.prompt, ...args } }),
        });
        return JSON.stringify({ status: "updated" });
      },
    },

    {
      name: "stage_reference",
      description: "Add a visual reference image to influence the stream",
      parameters: { image_url: "string", scale: "number?" },
      async execute(args) {
        if (!streamIdRef.current) return JSON.stringify({ error: "No active stream" });
        await fetch(`${sdkUrl}/stream/${streamIdRef.current}/control`, {
          method: "POST", headers: headers(),
          body: JSON.stringify({ type: "parameters", params: {
            vace_enabled: true,
            vace_ref_images: [args.image_url],
            vace_context_scale: args.scale ?? 0.8,
          }}),
        });
        return JSON.stringify({ status: "reference_applied" });
      },
    },

    {
      name: "stage_style",
      description: "Load a style (LoRA) into the live pipeline",
      parameters: { lora_path: "string", scale: "number?" },
      async execute(args) { /* scope_control with lora_scales */ },
    },

    {
      name: "stage_sync",
      description: "Sync a visual parameter to the music beat",
      parameters: { param: "string", wave: "string?", rate: "string?", depth: "number?" },
      async execute(args) {
        if (!streamIdRef.current) return JSON.stringify({ error: "No active stream" });
        await fetch(`${sdkUrl}/stream/${streamIdRef.current}/control`, {
          method: "POST", headers: headers(),
          body: JSON.stringify({ type: "parameters", params: {
            modulation: {
              [args.param || "noise_scale"]: {
                enabled: true,
                shape: args.wave || "cosine",
                rate: args.rate || "bar",
                depth: args.depth ?? 0.3,
              },
            },
          }}),
        });
        return JSON.stringify({ status: "modulation_set" });
      },
    },

    {
      name: "stage_record",
      description: "Start or stop recording the live stream",
      parameters: { action: "string" }, // "start" or "stop"
      async execute(args) {
        if (!streamIdRef.current) return JSON.stringify({ error: "No active stream" });
        await fetch(`${sdkUrl}/stream/${streamIdRef.current}/control`, {
          method: "POST", headers: headers(),
          body: JSON.stringify({ type: "parameters", params: { recording: args.action === "start" } }),
        });
        return JSON.stringify({ status: args.action === "start" ? "recording" : "stopped" });
      },
    },
  ];
}
```

Every tool calls the SDK's `/stream/*` endpoints. No direct Scope backend calls. Everything goes through the Livepeer network.

## 6. ScopePlayer — Dual Mode Component

```tsx
// packages/scope-player/src/ScopePlayer.tsx

interface ScopePlayerProps {
  // SDK mode (primary — remote Livepeer)
  mode?: "sdk" | "local";
  sdkUrl?: string;            // "https://sdk.daydream.monster"
  apiKey?: string;             // Daydream API key
  streamId?: string;           // From stage_start tool

  // Local mode (secondary — development)
  scopeUrl?: string;           // "http://localhost:8000"

  // Shared
  className?: string;
  onFps?: (fps: number) => void;
  onStateChange?: (state: "idle" | "connecting" | "streaming" | "error") => void;
  children?: ReactNode;       // Overlay content (scene name, recording indicator)
}

export function ScopePlayer(props: ScopePlayerProps) {
  if (props.mode === "local" && props.scopeUrl) {
    return <LocalScopePlayer {...props} />;
  }
  // Default: SDK trickle mode
  return <SdkScopePlayer {...props} />;
}
```

**SdkScopePlayer** uses `TrickleVideoPlayer` (canvas-based frame polling).
**LocalScopePlayer** uses WebRTC (extracted from Scope frontend).

## 9. How Scope's LivepeerClient Fits

**File:** `Scope/scope/src/scope/server/livepeer_client.py`

Scope's `LivepeerClient` is a **Python class inside the fal runner** — not a JavaScript client. It manages:
- Trickle channel connections (MediaPublish for input, MediaOutput for output)
- JSONLWriter for control channel (`send_parameters()`)
- Job lifecycle (`start_media()`, `stop_media()`)
- Output frame callbacks (FrameOutputHandler, AudioOutputHandler)
- Payment cycling (via livepeer_gateway)

**Creative Stage does NOT call LivepeerClient directly.** The full chain:

```
Creative Stage (browser)
  → SDK Service /stream/* (Python FastAPI on sdk.daydream.monster)
    → BYOC Orch (go-livepeer on byoc-staging-1)
      → Scope Orch (go-livepeer on orch-staging-1/2)
        → fal Scope runner (wss://fal.run/daydream/scope-livepeer/ws)
          → LivepeerClient.start_media(initial_parameters)
          → Scope backend spawns inside fal container
          → longlive pipeline on GPU
          → LivepeerClient.output_handlers[0] receives frames
        ← Trickle output channel
      ← Trickle output → SDK polls
    ← /stream/{id}/frame returns JPEG to browser
```

**Key LivepeerClient methods (internal to fal runner — NOT called by browser):**
- `start_media(initial_parameters)` — opens trickle channels with graph, prompts, pipeline config
- `send_parameters(params)` — writes `{ type: "parameters", params }` to control channel
- `stop_media()` — closes channels, job cleanup
- `output_handlers[0].on_frame(frame)` — callback when output frame is ready
- `audio_output_handler.on_audio(data)` — callback for audio output

**What this means for Creative Stage:**
The SDK's `/stream/*` endpoints are the correct API. They already wrap the entire LivepeerClient → fal runner → trickle protocol flow. Creative Stage calls:
- `POST /stream/start` → triggers `LivepeerClient.start_media()`
- `POST /stream/{id}/control` → triggers `LivepeerClient.send_parameters()`
- `GET /stream/{id}/frame` → reads from `LivepeerClient.output_handlers[0]`
- `POST /stream/{id}/stop` → triggers `LivepeerClient.stop_media()`

No new server-side code needed. The SDK is the bridge.

**Future: Audio streaming**
`LivepeerClient.audio_output_handler` captures audio output from the pipeline, but the SDK doesn't expose it over HTTP yet. When it does (new `/stream/{id}/audio` endpoint), Creative Stage gets real stream audio for free — just poll the new endpoint alongside frame polling.
